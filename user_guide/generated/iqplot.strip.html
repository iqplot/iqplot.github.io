Make a strip plot.

Parameters:

  • data (Pandas DataFrame, 1D Numpy array, or xarray) ‚Äì DataFrame containing tidy data for plotting. If a Numpy array, a single category is assumed and a strip plot generated from data.

  • q (hashable) ‚Äì Name of column to use as quantitative variable if data is a Pandas DataFrame. Otherwise, q is used as the quantitative axis label.

  • cats (hashable or list of hashables) ‚Äì Name of column(s) to use as categorical variable(s).

  • q_axis (str, either 'x' or 'y', default 'x') ‚Äì Axis along which the quantitative value varies.

  • palette (list of strings of hex colors, or single hex string) ‚Äì If a list, color palette to use. If a single string representing a hex color, all glyphs are colored with that color. Default is colorcet.b_glasbey_category10 from the colorcet package.

  • order (list or None) ‚Äì If not None, must be a list of unique group names when the input data frame is grouped by cats. The order of the list specifies the ordering of the categorical variables on the categorical axis and legend. If None, the categories appear in the order in which they appeared in the inputted data frame.

  • p (bokeh.plotting.Figure instance, or None (default)) ‚Äì If None, create a new figure. Otherwise, populate the existing figure p.

  • show_legend (bool, default False) ‚Äì If True, display legend.

  • legend_location (str, default 'right') ‚Äì Location of legend. If one of ‚Äúright‚Äù, ‚Äúleft‚Äù, ‚Äúabove‚Äù, or ‚Äúbelow‚Äù, the legend is placed outside of the plot area. If one of ‚Äútop_left‚Äù, ‚Äútop_center‚Äù, ‚Äútop_right‚Äù, ‚Äúcenter_right‚Äù, ‚Äúbottom_right‚Äù, ‚Äúbottom_center‚Äù, ‚Äúbottom_left‚Äù, ‚Äúcenter_left‚Äù, or ‚Äúcenter‚Äù, the legend is placed within the plot area. If a 2-tuple, legend is placed according to the coordinates in the tuple.

  • legend_orientation (str, default 'vertical') ‚Äì Either ‚Äòhorizontal‚Äô or ‚Äòvertical‚Äô.

  • legend_click_policy (str, default 'hide') ‚Äì Either ‚Äòhide‚Äô, ‚Äòmute‚Äô, or None; how the glyphs respond when the corresponding category is clicked in the legend.

  • color_column (hashable, default None) ‚Äì Column of data to use in determining color of glyphs. If None, then cats is used.

  • parcoord_column (hashable, default None) ‚Äì Column of data to use to construct a parallel coordinate plot. Data points with like entries in the parcoord_column are connected with lines.

  • tooltips (list of 2-tuples) ‚Äì Specification for tooltips as per Bokeh specifications. For example, if we want col1 and col2 tooltips, we can use tooltips=[(‚Äòlabel 1‚Äô: ‚Äò@col1‚Äô), (‚Äòlabel 2‚Äô: ‚Äò@col2‚Äô)].

  • marker (str, default 'circle') ‚Äì Name of marker to be used in the plot (ignored if formal is False). Must be one of[‚Äòasterisk‚Äô, ‚Äòcircle‚Äô, ‚Äòcircle_cross‚Äô, ‚Äòcircle_x‚Äô, ‚Äòcross‚Äô, ‚Äòdash‚Äô, ‚Äòdiamond‚Äô, ‚Äòdiamond_cross‚Äô, ‚Äòhex‚Äô, ‚Äòinverted_triangle‚Äô, ‚Äòsquare‚Äô, ‚Äòsquare_cross‚Äô, ‚Äòsquare_x‚Äô, ‚Äòtriangle‚Äô, ‚Äòx‚Äô]

  • spread (str or None, default None) ‚Äì If ‚Äòjitter‚Äô, spread points out using a jitter transform. If ‚Äòswarm‚Äô, spread points in beeswarm style. In None or ‚Äònone‚Äô, do not spread.

  • cat_grid (bool, default False) ‚Äì If True, show grid lines for categorical axis.

  • marker_kwargs (dict) ‚Äì Keyword arguments to pass when adding markers to the plot. [‚Äúx‚Äù, ‚Äúy‚Äù, ‚Äúsource‚Äù, ‚Äúcat‚Äù, ‚Äúlegend‚Äù] are note allowed because they are determined by other inputs.

  • jitter_kwargs (dict) ‚Äì Keyword arguments to be passed to bokeh.transform.jitter(). If not specified, default is {‚Äòdistribution‚Äô: ‚Äònormal‚Äô, ‚Äòwidth‚Äô: 0.1}. If the user specifies {‚Äòdistribution‚Äô: ‚Äòuniform‚Äô}, the ‚Äòwidth‚Äô entry is adjusted to 0.4.

  • swarm_kwargs (dict) ‚Äì

    Keyword arguments for use in generating swarm. Keys with allowed values are:

    • ‚Äôcorral‚Äô: Either ‚Äògutter‚Äô (default) or ‚Äòwrap‚Äô. This

    specifies how points that are moved too far out are dealt with. Using ‚Äògutter‚Äô, points are overlayed at the maximum allowed distance. Using ‚Äòwrap‚Äô, points are reflected inwards form the maximal extent and possibly overlayed with other points.

  • horizontal (bool or None, default None) ‚Äì Deprecated. Use q_axis.

  • val (hashable) ‚Äì Deprecated, use q.

  • click_policy (str, default 'hide') ‚Äì Deprecated. Use legend_click_policy.

  • kwargs ‚Äì Any kwargs to be passed to bokeh.plotting.figure() when instantiating the figure.
